# Portfolio Project: RAG Q&A

## Objective

By the end of this project, you will ship a grounded QA system with citations and monitoring.

## Prerequisites

Before this project, you should complete all Module 07 lessons and mini project.

## Plain-English Explanation

Build a user-facing QA feature backed by documents and evidence.

## Key Terms

### Monitoring

Simple meaning:
Keep an eye on behavior over time.

Real-world analogy:
Track support SLA and quality every day.

Technical meaning:
Measure retrieval quality, grounding, and latency continuously.

Example:
Dashboard with retrieval precision and citation ratio.

Why it matters:
RAG systems decay without observability.

## Mental Model

Ingest → index → query → retrieve → rank → generate → validate → report.

## Example

Create a product FAQ Q&A feature with source highlights and review queue.

## Code Walkthrough

Build ingestion script, query endpoint, and evaluation logs.

## Common Mistakes

- Missing update jobs for embeddings.
- Infrequent evaluation.
- No trace for cited outputs.

## Exercise

Publish a launch checklist and three measurable SLOs.

## Reflection

What will you do if citations are consistently weak?

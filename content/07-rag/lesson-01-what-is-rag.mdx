# What is RAG?

## Objective

By the end of this lesson, you will define RAG and explain where it fits in applications.

## Prerequisites

Before this lesson, you should complete Module 06 LLM Engineering.

## Plain-English Explanation

RAG retrieves relevant evidence and gives it to the model before it answers.

## Key Terms

### Grounding

Simple meaning:
Anchoring the answer in source material.

Real-world analogy:
Writing a legal response with statutes in front of you.

Technical meaning:
Attaching retrieved context to generation inputs.

Example:
Lookup policy docs before generating customer guidance.

Why it matters:
Grounding makes outputs more credible and auditable.

## Mental Model

Think of RAG as “search → cite → answer.”

## Example

Customer support bot pulling relevant policy articles before responding.

## Code Walkthrough

Build a flow with retrieval call, passage injection, and final synthesis.

## Common Mistakes

- Skipping citation checks.
- Returning claims without source.
- Using stale indexes.

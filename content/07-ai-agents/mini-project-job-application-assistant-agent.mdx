# Mini Project: Job Application Assistant Agent

## Objective

By the end of this mini project, you can build a constrained multi-step agent with review gates.

## Prerequisites

Before this mini project, complete all Module 07 lessons through multi-step workflows.

## Plain-English Explanation

Create an assistant that discovers opportunities, drafts content, and pauses for approval before submission.

## Key Terms

### Escalation

Simple meaning:
When the system asks a human for permission.

Real-world analogy:
Manager approval before publishing a campaign.

Technical meaning:
Workflow transition to manual review path with context attached.

Example:
Draft generated by low-confidence rule.

Why it matters:
Escalation protects risky actions while keeping speed.

## Mental Model

User goal -> role lookup -> draft generation -> approval check -> submission queue.

## Example

The agent prepares a tailored application and holds it until user confirms.

## Code Walkthrough

Create:
- Tool schema for role lookup and draft creation
- Approval queue state
- Finalization action

## Common Mistakes

- Sending work without explicit approval.
- Losing conversation context after approval.
- No summary of changes in approval payload.

## Exercise

Build one escalation condition and demonstrate it in writing.

# Human Approval

## Objective

By the end of this lesson, you can design handoff points where humans remain in control.

## Prerequisites

Before this lesson, you should complete Module 07 Lessons 01–05.

## Plain-English Explanation

Some actions are better reviewed by people before execution.

## Key Terms

### Human-in-the-Loop

Simple meaning:
People approve high-impact actions.

Real-world analogy:
An editor approving a generated press release.

Technical meaning:
Policy gate that pauses workflow until explicit consent.

Example:
An agent can draft but not send email without review.

Why it matters:
Keeps trust and reduces risk for sensitive operations.

## Mental Model

Think of the loop as "pause points" inserted at risk boundaries.

## Example

Route outbound applications to review only when confidence is low.

## Code Walkthrough

Add an approval record with payload, approver_id, timestamp, and decision.

## Common Mistakes

- Making human review mandatory for every action.
- Missing context in approval prompt.
- No audit log for decisions.

## Exercise

Create two categories of actions: automatic vs review-required.

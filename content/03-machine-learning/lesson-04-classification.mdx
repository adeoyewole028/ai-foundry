# Classification

## Objective

By the end of this lesson, you will choose between class labels and probability outputs.

## Prerequisites

Before this lesson, you should understand splitting and what a target variable is.

## Plain-English Explanation

Classification decides which bucket an input belongs to.

## Key Terms

### Class

Simple meaning:
A category like “hired” or “not hired.”

Real-world analogy:
Sorting emails into “spam” and “not spam.”

Technical meaning:
Supervised learning where output domain is discrete.

Example:
Fraud/no-fraud transaction flags.

Why it matters:
Most business decisions are category based and must handle imbalance.

## Mental Model

Visualize a boundary separating categories in feature space.

## Example

Predict whether a candidate is a likely match based on profile strength and interest history.

## Code Walkthrough

Train a classifier baseline and compare confusion matrix outputs before finalizing a threshold.

## Common Mistakes

- Ignoring class imbalance.
- Reporting only accuracy.
- Setting thresholds after seeing test labels.

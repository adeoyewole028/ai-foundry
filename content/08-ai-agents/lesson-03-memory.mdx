# Agent Memory

## Objective

By the end of this lesson, you can design memory strategies that balance continuity and safety.

## Prerequisites

Before this lesson, you should complete Lessons 01 and 02 in this module.

## Plain-English Explanation

Memory helps an agent remember purpose, constraints, and prior actions.

## Key Terms

### Persistent Memory

Simple meaning:
Information stored across sessions.

Real-world analogy:
Customer profile kept in CRM.

Technical meaning:
State and artifacts saved to external storage for reuse.

Example:
Saved job applications and review status.

Why it matters:
Without memory, agents repeat work and lose context.

## Mental Model

Imagine memory as two layers: immediate notes for one run and durable records for the journey.

## Example

An agent keeps the user's role preference while switching from discovery to draft mode.

## Code Walkthrough

Track: `goal`, `latest_state`, `blocked_items`, and `handoff_reason`.

## Common Mistakes

- Storing too much raw context.
- Retaining sensitive data too broadly.
- No retention policy.

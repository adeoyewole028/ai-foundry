# What is an AI Agent?

## Objective

By the end of this lesson, you can explain when autonomy helps versus when deterministic logic is better.

## Prerequisites

Before this lesson, you should complete Modules 00–06 and understand tool invocation basics.

## Plain-English Explanation

An agent decides a goal-oriented sequence of actions instead of giving only one response.

## Key Terms

### Autonomy

Simple meaning:
The system acts with permission rules and plans next actions.

Real-world analogy:
Like a smart assistant that books flights, then sends reminders, then checks status.

Technical meaning:
An orchestrator loop that reasons, executes tools, and updates state.

Example:
Gather role data, draft cover letter, and queue revision steps.

Why it matters:
Autonomy can multiply productivity, but needs safeguards.

## Mental Model

Think of the agent as a pilot: it has destination, instruments, and rules for corrections.

## Example

Instead of one-turn answers, the agent checks job sources, drafts matching content, and asks approval before sending.

## Code Walkthrough

Model an agent loop as: goal -> plan -> execute -> observe -> update -> terminate.

## Common Mistakes

- Giving the agent unrestricted execution without limits.
- Ignoring noisy state changes across turns.
- No explicit termination condition.

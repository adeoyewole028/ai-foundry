## Plain-English Explanation

Build a reliable assistant that moves applications from discovery to reviewed draft and tracks execution history.

## Key Terms

### Audit Trail

Simple meaning:
Record of what happened and why.

Real-world analogy:
Version history in a legal document workflow.

Technical meaning:
Persistent logs for actions, approvals, outcomes, and failures.

Example:
Storing each approval request with timestamp and state snapshot.

Why it matters:
Audits support debugging, trust, and iteration.

## Mental Model

Intake -> Agent plan -> Tool run -> Human checkpoint -> Finalize -> Archive.

## Example

From one job posting URL to final ready-to-send application package.

## Code Walkthrough

Implement:
- Plan and state data models
- Tool wrappers with strict schemas
- Dashboard metrics for completion and escalation

## Common Mistakes

- No rollback on partial success.
- Hardcoded tool calls.
- Missing user messaging at each handoff.

# Tools for Agents

## Objective

By the end of this lesson, you can design safe and typed tools for autonomous execution.

## Prerequisites

Before this lesson, you should finish Module 06 on tool calling and this module’s first lesson.

## Plain-English Explanation

Tools are controlled functions that limit what the agent can do.

## Key Terms

### Tool Contract

Simple meaning:
The agreed input/output for an action.

Real-world analogy:
An office SOP: what form to submit, what to receive.

Technical meaning:
Machine-readable schema for arguments, return values, and side effects.

Example:
`search_jobs({keywords, location}) -> jobs[]`.

Why it matters:
Contracts stop ambiguous or dangerous calls.

## Mental Model

Treat each tool as a locked API door: the agent can only act through known doors.

## Example

The agent may call `draft_summary`, `send_draft_for_review`, or `save_to_history`.

## Code Walkthrough

For each tool define:
- required and optional args
- validation checks
- error response shape

## Common Mistakes

- Tools with broad side effects and weak validation.
- Missing structured errors.
- No per-tool audit logs.

## Exercise

Write a tool contract for an application that uploads resumes and returns parsed fields.

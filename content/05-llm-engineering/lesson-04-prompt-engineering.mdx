# Prompt Engineering

## Objective

By the end of this lesson, you will write prompts that produce dependable outputs.

## Prerequisites

Before this lesson, you should understand context and tokens.

## Plain-English Explanation

A good prompt sets role, goal, constraints, and output format clearly.

## Key Terms

### Instruction hierarchy

Simple meaning:
Which rule overrides which when prompts conflict.

Real-world analogy:
Company policy overrides team preference.

Technical meaning:
Precedence between system, developer, and user instructions.

Example:
System says “be strict,” user says “be funny.”

Why it matters:
Conflict handling determines final behavior.

## Mental Model

Stack prompts from highest intent to lowest.

## Example

Build an FAQ assistant with one strict response structure.

## Code Walkthrough

Compare two versions and log answer quality.

## Common Mistakes

- Vague goals and ambiguous formatting.
- Asking for many constraints in one sentence.
- Not including output schema.

## Exercise

Create three versions of one prompt and measure consistency.

## Reflection

What makes prompt behavior predictable across reruns?

# Python Setup

## Objective

By the end of this lesson, you will be able to set up Python and run reproducible scripts from a clean workspace.

## Prerequisites

No prior Python knowledge required.

## Plain-English Explanation

Python setup is choosing the right version, creating a project folder, and avoiding hidden environment traps. The goal is simple: when you run code today, it should run the same way tomorrow.

## Key Terms

### Interpreter

Simple meaning:
The Python interpreter executes your script.

Real-world analogy:
Think of it as a translator that reads your instructions and performs each command.

Technical meaning:
The installed Python runtime selected to execute your code.

Example:
Running `python` in a terminal opens that interpreter.

Why it matters:
Different runtimes can lead to different behavior if versions differ.

## Mental Model

```text
Install Python -> Create environment -> Run script -> Verify output
```

## Example

Run:

```bash
python -V
python -m venv .venv
```

## Code Walkthrough

```bash
# create and use an isolated environment
python -m venv .venv
source .venv/bin/activate
python -V
```

## Common Mistakes

- Installing dependencies globally and inheriting unknown versions.
- Running `python` and `pip` from different environments.

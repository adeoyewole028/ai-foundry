# Virtual Environments

## Objective

By the end of this lesson, you will keep Python projects reproducible with isolated dependencies.

## Prerequisites

API and files lessons.

## Plain-English Explanation

Without isolated environments, dependency versions drift and your teammate's working state can diverge. Virtual environments keep projects self-contained.

## Key Terms

### Dependency

Simple meaning:
An external package your code depends on.

Real-world analogy:
Like ingredients for a recipe that must be in the right amounts.

Technical meaning:
Python package installed via `pip` in an environment.

Example:
`requests==2.31.0` pinned in `requirements.txt`.

Why it matters:
Dependency drift causes mysterious runtime errors.

## Mental Model

```text
Project -> requirements -> environment -> predictable runs
```

## Example

```bash
python -m venv .venv
source .venv/bin/activate
pip install requests==2.31.0
pip freeze > requirements.txt
```

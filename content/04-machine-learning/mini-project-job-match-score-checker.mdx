# Mini Project: Job Match Score Checker

## Objective

By the end of this project, you will prototype a robust scoring workflow with a clean split strategy.

## Prerequisites

## Plain-English Explanation

This mini project checks if your scoring logic can separate promising candidates from weak matches without leakage.

## Key Terms

### Baseline model

Simple meaning:
A simple starting method before advanced improvements.

Real-world analogy:
A rough draft before final copy.

Technical meaning:
A minimal predictive model with explicit assumptions and transparent metrics.

Example:
Rule-based + simple classifier blend for initial scoring.

Why it matters:
Baselines prevent over-building before validation confirms value.

## Mental Model

Build a narrow pipeline: ingest → split → baseline → evaluate → compare.

## Example

Train on historical candidate outcomes and validate how stable scores are across cohorts.

## Code Walkthrough

Create a notebook or script with:

- Data schema check
- Feature/label split
- Train/validation split with seed
- One baseline model
- One metric report

## Common Mistakes

- Evaluating only on training data.
- Not tracking random seed.
- Changing pipeline without logging results.

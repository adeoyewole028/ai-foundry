# CSV Files

## Objective

By the end of this lesson, you will be able to load and inspect CSV data with predictable checks.

## Prerequisites

Before this lesson, you should know how to run Python scripts and inspect file content.

## Plain-English Explanation

CSV files are simple tables in text form. They look easy, but small format changes break parsing.

## Key Terms

### Delimiter

Simple meaning:
The character that separates values in each row (often comma).

Real-world analogy:
Like choosing commas to separate items in a shopping list.

Technical meaning:
Different delimiters (``,``, `;`, `|`) and quoting rules change parsing behavior.

Example:
A semicolon-separated file opened as comma-separated will produce extra columns or misreads.

Why it matters:
Parsing errors can quietly corrupt your training source.

## Mental Model

Treat CSV ingestion as a contract. Before reading rows, define and verify headers, delimiter, encoding, and null markers.

## Example

If a file uses `,` in text fields, proper quoting avoids column shifts.

## Code Walkthrough

Read a file, verify `columns`, `shape`, and a few sample rows before downstream processing.

## Common Mistakes

- Trusting row 0 as perfect schema.
- Ignoring BOM/encoding warnings.
- Skipping null checks and proceeding as if nothing is missing.
